# What is a JOIN in SQL?

A JOIN is used to combine rows from two or more tables based on a related column (usually a primary key and foreign key).


# Types of Joins

| JOIN Type  | Returns                   |
| ---------- | ------------------------- |
| INNER JOIN | Only matching rows        |
| LEFT JOIN  | All left + matching right |
| RIGHT JOIN | All right + matching left |
| FULL JOIN  | All rows from both tables |
| CROSS JOIN | All combinations          |
| SELF JOIN  | Table joined to itself    |

## **1. INNER JOIN**

An INNER JOIN is used to retrieve only the records that have matching values in both tables based on a specified condition.

## Example Questions

**Q1. List employee name and department name for employees who belong to a department.**

```
SELECT e.emp_name, d.dept_name
FROM employees3 e
INNER JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q2. Find employees working in the IT department.**

```
SELECT e.emp_name, d.dept_name
FROM employees3 e
INNER JOIN departments3 d
ON e.dept_id = d.dept_id
WHERE d.dept_name = 'IT';
```

**Q3. Show employee name, salary, and department location.**

```
SELECT e.emp_name, e.salary, d.location
FROM employees3 e
INNER JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q4. List employees whose department is located in Chennai.**

```
SELECT e.emp_name, d.location
FROM employees3 e
INNER JOIN departments3 d
ON e.dept_id = d.dept_id
WHERE d.location = 'Chennai';
```

**Q5. Count number of employees in each department.**

```
SELECT d.dept_name, COUNT(e.emp_id) AS total_employees
FROM employees3 e
INNER JOIN departments3 d
ON e.dept_id = d.dept_id
GROUP BY d.dept_name;
```

## **2. LEFT JOIN**

A LEFT JOIN (also called LEFT OUTER JOIN) returns all records from the left table and only the matching records from the right table.

If there is no match, the result will contain NULL values for columns from the right table.

**Q1. List all employees with their department name (include employees without department).**

```
SELECT e.emp_name, d.dept_name
FROM employees3 e
LEFT JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q2. Find employees who do NOT belong to any department.**

```
SELECT e.emp_name
FROM employees3 e
LEFT JOIN departments3 d
ON e.dept_id = d.dept_id
WHERE d.dept_id IS NULL;
```  

**Q3. Show employee name, salary, and department location (even if NULL).**

```
SELECT e.emp_name, e.salary, d.location
FROM employees3 e
LEFT JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q4. List all employees and departments; show NULL where department missing.**

```
SELECT e.emp_id, e.emp_name, d.dept_name
FROM employees3 e
LEFT JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q5. Count employees per department including unassigned employees.**

```
SELECT d.dept_name, COUNT(e.emp_id) AS total_employees
FROM employees3 e
LEFT JOIN departments3 d
ON e.dept_id = d.dept_id
GROUP BY d.dept_name;
```

## 3. RIGHT JOIN

A RIGHT JOIN (also called RIGHT OUTER JOIN) returns all records from the right table and only the matching records from the left table.

If there is no match, the result will contain NULL values for columns from the left table.

**Q1. List all departments with their employees.**

```
SELECT d.dept_name, e.emp_name
FROM employees3 e
RIGHT JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q2. Find departments with no employees.**

```
SELECT d.dept_name
FROM employees3 e
RIGHT JOIN departments3 d
ON e.dept_id = d.dept_id
WHERE e.emp_id IS NULL;
```

**Q3. Show department name and employee salary.**

```
SELECT d.dept_name, e.salary
FROM employees3 e
RIGHT JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q4. Count employees in each department (including zero).**

```
SELECT d.dept_name, COUNT(e.emp_id) AS total_employees
FROM employees3 e
RIGHT JOIN departments3 d
ON e.dept_id = d.dept_id
GROUP BY d.dept_name;
```

**Q5. List all department locations with employee names.**

```
SELECT d.location, e.emp_name
FROM employees3 e
RIGHT JOIN departments3 d
ON e.dept_id = d.dept_id;
```

## 4. FULL JOIN

**Q1. List all employees and all departments.**

```
SELECT e.emp_name, d.dept_name
FROM employees3 e
FULL OUTER JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q2. Find employees without department and departments without employees.**

```
SELECT e.emp_name, d.dept_name
FROM employees3 e
FULL OUTER JOIN departments3 d
ON e.dept_id = d.dept_id
WHERE e.emp_id IS NULL OR d.dept_id IS NULL;
```

**Q3. Show employee salary and department location.**

```
SELECT e.emp_name, e.salary, d.location
FROM employees3 e
FULL OUTER JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q4. Count total rows returned by FULL JOIN.**

```
SELECT COUNT(*)
FROM employees3 e
FULL OUTER JOIN departments3 d
ON e.dept_id = d.dept_id;
```

**Q5. Display unmatched records clearly.**

```
SELECT
  COALESCE(e.emp_name, 'No Employee') AS emp_name,
  COALESCE(d.dept_name, 'No Department') AS dept_name
FROM employees3 e
FULL OUTER JOIN departments3 d
ON e.dept_id = d.dept_id;
```
